# 06 阶段三执行文档

## 1. 阶段目标

阶段三的目标是把 CRM 客户扩展信息正式接入合同和回款流程，形成下面这条业务链：

1. 客户维护银行账户、营业执照、资质
2. 合同签约时选择默认收款账户
3. 回款登记时默认带出合同收款账户
4. 业务单详情只读展示合同和回款摘要，不重复承载财务流程

本阶段仍然坚持边界不变：

- `freight_order` 只负责履约
- `crm_contract` 负责合同
- `crm_receivable_plan` 负责回款计划
- `crm_receivable` 负责实际回款
- `crm_customer_bank_account` 负责客户银行账户
- `crm_customer_license` 负责客户证照与资质

## 2. 交付范围

### 2.1 合同模块

- 合同增加 `bankAccountId`
- 合同表单支持按客户选择收款账户
- 合同详情展示收款账户摘要
- 合同变更客户时自动清空不匹配的账户

### 2.2 回款模块

- 回款增加 `bankAccountId`
- 新建回款时默认回填合同上的收款账户
- 允许切换为该客户名下其他银行账户
- 回款详情展示收款账户信息

### 2.3 客户模块

- 客户详情页继续维护银行账户和资质
- 客户资质增加状态识别：
  - 有效
  - 即将到期
  - 已过期

### 2.4 业务单模块

- 业务单详情页增加合同收款账户只读展示
- 不新增任何回款主表、回款计划主表、合同主表

## 3. 数据库改造

### 3.1 合同表增加收款账户关联

目标表：`crm_contract`

新增字段建议：

- `bank_account_no`

说明：

- `bank_account_id` 作为主关联字段

### 3.2 回款表增加收款账户关联

## 4. 后端改造清单

### 4.1 合同模块

建议改造点：

1. `CrmContractDO`
   - 增加 `bankAccountId`
   - 增加快照字段
2. 合同 SaveReqVO / RespVO
   - 同步新增字段
3. 合同 Service
   - 保存合同时校验银行账户存在
   - 校验该账户属于合同客户
   - 自动回填快照字段
4. 合同详情聚合
   - 增加收款账户信息输出

建议新增错误码：

- 合同关联银行账户不存在
- 银行账户不属于当前客户

### 4.2 回款模块

建议改造点：

1. `CrmReceivableDO`
   - 增加 `bankAccountId`
   - 增加快照字段
2. 回款 SaveReqVO / RespVO
   - 同步新增字段
3. 回款 Service
   - 保存回款时校验账户归属客户
   - 如果传入 `contractId` 且未传 `bankAccountId`，优先带出合同默认账户
   - 如果合同已有默认账户但请求带入其他账户，需要校验该账户仍属于同一客户
4. 回款详情
   - 输出收款账户摘要

### 4.3 客户资质状态计算

建议在后端或前端统一实现状态识别规则：

- `expireDate` 为空：显示 `长期有效`
- `expireDate >= 今天 + 30 天`：显示 `有效`
- `今天 <= expireDate < 今天 + 30 天`：显示 `即将到期`
- `expireDate < 今天`：显示 `已过期`

建议在返回 VO 中增加一个可选展示字段：

- `expireStatus`

如果不想改后端 VO，也可以先在前端计算。

## 5. 前端改造清单

### 5.1 合同表单

页面：

- CRM 合同新建/编辑弹窗

改造点：

1. 增加 `收款账户` 字段
2. 先选客户，再按客户加载银行账户列表
3. 切换客户时清空不匹配账户
4. 选中后在表单中显示：
   - 户名
   - 开户行
   - 银行账号后四位或完整账号

### 5.2 合同详情

页面：

- CRM 合同详情页

新增展示区：

- 收款账户信息
  - 户名
  - 开户行
  - 银行账号

同时保留：

- 回款计划
- 实际回款
- 关联业务单

### 5.3 回款表单

页面：

- CRM 回款新建/编辑弹窗

改造点：

1. 增加 `收款账户` 字段
2. 如果选择了合同，则自动带出合同默认账户
3. 允许手工切换当前客户其他账户
4. 切换合同后重新校验账户是否仍匹配

### 5.4 客户详情资质页

当前已有：

- 银行账户 Tab
- 营业执照/资质 Tab

本阶段建议增强：

1. 资质列表增加状态列
2. 用颜色区分：
   - 绿色：有效
   - 橙色：即将到期
   - 红色：已过期
3. 支持默认按到期时间排序

### 5.5 业务单详情页

页面：

- 国际货代业务单详情页

新增合同财务摘要展示项：

- 合同编号
- 合同金额
- 已回款
- 未回款
- 默认收款账户

说明：

- 业务单仍然不编辑收款账户
- 业务单只做只读展示

## 6. 接口建议

### 6.1 合同相关

复用已有合同 CRUD 接口，在 VO 中扩字段即可。

如需前端下拉选择方便，可以增加：

- `GET /crm/customer-bank-account/list-by-customer?customerId=`

该接口已经在上一阶段存在，可直接复用。

### 6.2 回款相关

复用已有回款 CRUD 接口，在 VO 中扩字段即可。

如果前端需要更顺滑，可以增加一个轻量接口：

- `GET /crm/contract/default-bank-account?id=`

返回：

- `bankAccountId`
- `bankAccountName`
- `bankName`
- `bankAccountNo`

这不是必须项。

如果合同详情接口已经返回该数据，也可以不额外加接口。

## 8. 验收标准

本阶段完成后，应满足以下验收项：

1. 客户详情页可维护多个银行账户
2. 客户详情页可维护多个营业执照/资质
3. 合同创建时可选择当前客户名下银行账户
4. 保存合同后，合同详情可查看收款账户摘要
5. 回款登记时可自动带出合同默认收款账户
6. 回款登记时允许切换到同客户其他账户
7. 不允许将其他客户的账户绑到当前合同或回款
8. 业务单详情只能展示合同与回款摘要，不新增回款录入

## 9. 风险与注意事项

### 9.1 历史数据兼容

- 旧合同没有 `bank_account_id`
- 旧回款没有 `bank_account_id`

要求：

- 字段允许为空
- 老数据可渐进补录

### 9.2 快照一致性

- 即使客户后续修改银行账户名称、开户行、账号，历史合同和回款仍应保留当时快照

### 9.3 删除限制

建议增加业务约束：

- 若某银行账户已被合同使用，不允许直接删除
- 若某银行账户已被回款使用，不允许直接删除

### 9.4 权限边界

建议权限分开：

- 客户资料维护
- 合同维护
- 回款登记
- 回款审核

## 10. 推荐实施结果

阶段三落地后，整体模型会变成：

1. CRM 客户维护主数据、银行账户、资质
2. CRM 合同绑定客户和默认收款账户
3. CRM 回款计划、回款围绕合同运转
4. Freight 业务单只承载国际货代履约过程

这样可以避免 freight 模块继续膨胀，同时又把客户、合同、回款链路真正打通。
