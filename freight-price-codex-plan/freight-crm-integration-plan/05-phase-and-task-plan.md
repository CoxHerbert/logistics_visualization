# 05 分期与任务清单

## 1. 一期目标

目标：先把 freight 与 CRM 打通，不在 freight 里重复造合同与回款。

### 交付范围

- `freight_order` 增加 `contract_id`
- 业务单表单支持合同选择
- 业务单详情展示合同摘要
- CRM 合同详情增加“关联业务单”

### 后端任务

1. 修改 `freight_order` 表
2. 修改 `FreightOrderDO / RespVO / SaveReqVO`
3. 保存业务单时校验合同合法性
4. 业务单详情聚合合同金额与回款摘要
5. 增加按合同查询业务单接口
6. 增加按客户查询合同精简列表接口

### 前端任务

1. 业务单表单增加合同下拉
2. 客户变化时重载合同列表
3. 业务单列表增加合同列
4. 业务单详情增加合同摘要区域
5. CRM 合同详情新增“关联业务单” Tab

## 2. 二期目标

目标：补 CRM 客户扩展资料。

### 交付范围

- 客户银行账户管理
- 客户营业执照 / 资质管理
- 客户详情页增加两个 Tab

### 后端任务

1. 新增 `crm_customer_bank_account`
2. 新增 `crm_customer_license`
3. 完成 DO / Mapper / Service / Controller
4. 完成权限点与菜单

### 前端任务

1. 客户详情页增加 `银行账户` Tab
2. 客户详情页增加 `营业执照/资质` Tab
3. 两个 Tab 都实现 Table + 弹窗 Form

## 3. 三期目标

目标：让合同、回款与客户扩展打通。

### 交付范围

- 合同选择默认收款账户
- 回款登记选择收款账户
- 客户资质摘要展示
- 证照到期提醒

## 4. 风险点

### 数据一致性

- 业务单客户与合同客户必须一致
- 删除合同前需校验是否已有业务单

### 历史数据兼容

- 历史业务单没有 `contract_id`
- 需要允许空值，并支持后续补录

### 菜单与路由

- 前端建议继续以后端菜单为主
- 隐藏详情页路由要在后端菜单里补齐，避免 404

## 5. 推荐开发顺序

1. 先做一期
2. 一期稳定后再做客户扩展
3. 最后再做账户与回款联动

这是风险最低、收益最高的落地顺序。
